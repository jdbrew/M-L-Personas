# Module Testing Criteria and Evaluation Questions
## Stage 4: Executive Managers and Leaders

**Version:** 1.0

---

## Document Purpose

This document contains evaluation questions and testing scenarios for assessing whether learning modules genuinely address power, difference, inclusion, and transformational learning as specified in the design guidelines. These criteria are derived from the diverse personas across all stages of the framework and support gap-analysis testing of module design and content.

## Usage Guidance

Use these questions and scenarios to test modules against:
- Design Principle 10 (Difference, diversity, inclusion, and power as golden thread)
- P + Q = L formula (transformational inquiry not just information)
- I-We-It integration
- Level-appropriate complexity
- Accessibility and inclusion
- Evidence gathering support
- Acknowledgment of limitations

## Personas

### Persona 23: Programme Director for Quality and Governance
**Diversity Context:** Mixed heritage (Black Caribbean/White British) woman, exec level, "only one"

#### Module Evaluation Questions
- Does module address leading as "only one" at Board level?
- Do examples acknowledge that executive leadership looks different for women of colour?
- Will this help me navigate representing without tokenising?
- Does it address the psychological cost of "firsts" and "onlys"?
- Does it help embed equity in quality and governance not just diversity statements?

#### Module Testing Scenarios

**Scenario 1: Respond to patient safety concerns, needs and preferences**
- **Test Question:** Does module address that "patient-centric" must include addressing health inequalities? Does it connect quality to equity?
- **Evaluation Criteria:**
  - Integrates equity into quality improvement
  - Addresses power dynamics in "patient voice"
  - Provides frameworks for culture change at scale

**Scenario 2: Implement policies and ensure good governance**
- **Test Question:** Does module help distinguish strategic from operational governance? Does it address influencing Board toward equity focus?
- **Evaluation Criteria:**
  - Clarifies strategic governance role
  - Addresses Board dynamics and influence
  - Connects governance to equity outcomes

**Scenario 3: Personalise care**
- **Test Question:** Does module address sustainability of strategic commitments? Does it help balance long-term transformation with immediate demands?
- **Evaluation Criteria:**
  - Addresses sustainability and long-term impact
  - Provides frameworks for strategic persistence
  - Acknowledges tension between transformation and operations

  - Helps maintain nursing focus in corporate priorities

- **Test Question:** Does module help sustain focus on improvement amidst operational pressures? Does it address creating space for innovation in constrained system?
- **Evaluation Criteria:**
  - Addresses sustainability of improvement focus
  - Provides frameworks for innovation in constrained contexts
  - Helps balance operational demands with strategic transformation

## Cross-Cutting Evaluation Themes

### Theme 1: Principle 10 - Power, Difference, and Inclusion
**Key Questions:**
- Does the module treat diversity as integral to competency, not separate topic?
- Does it surface who has power, whose knowledge is privileged, whose voices are heard?
- Does it address structural inequalities, not just individual behaviour?
- Does it help participants recognise their own positionality and privilege?
- Does it provide frameworks for action, not just awareness?

### Theme 2: Transformational Inquiry (Q component)
**Key Questions:**
- Does the module provoke genuine questioning of assumptions?
- Does it create discomfort that leads to transformed understanding?
- Is there space for critical reflection on power and practice?
- Does it challenge rather than confirm existing thinking?
- Does it move beyond information transfer to transformation?

### Theme 3: Accessibility and Universal Design
**Key Questions:**
- Does the module work for diverse learning needs and access requirements?
- Are examples and language accessible across educational backgrounds?
- Does it assume standard working patterns and availability?
- Can it be accessed flexibly across time zones, shifts, caring responsibilities?
- Does it provide multiple entry points and formats?

### Theme 4: Contextualisation and Application
**Key Questions:**
- Does the module prompt participants to translate to their specific context?
- Does it recognise diverse NHS roles, settings, and contexts?
- Does it address practical constraints and systemic pressures?
- Does it avoid generic examples that assume particular contexts?
- Does it help participants navigate their unique situation?

### Theme 5: Evidence Gathering and Competency Assessment
**Key Questions:**
- Does the module clearly guide what evidence demonstrates competency?
- Does it acknowledge completion â‰  competency achievement?
- Does it provide realistic timeframes for evidence gathering?
- Does it direct toward formal assessment processes?
- Does it support ongoing practice-based evidence collection?

### Theme 6: Level-Appropriate Complexity
**Key Questions:**
- Is conceptual complexity appropriate for the stage?
- Does scope match organisational perspective (team/service/system)?
- Is balance of guidance vs exploration appropriate?
- Does it respect developmental readiness?
- Does it avoid patronising or overwhelming?

### Theme 7: Integration Across I-We-It
**Key Questions:**
- Does module address personal mastery (I) dimension?
- Does it address relational capability (We) dimension?
- Does it address operational effectiveness (It) dimension?
- Does it show how these integrate rather than separate?
- Does it avoid fragmented skill development?

### Theme 8: Acknowledgment of Limitations
**Key Questions:**
- Does module explicitly state what it cannot achieve?
- Does it acknowledge 2-hour engagement cannot complete development?
- Does it direct toward deeper learning opportunities?
- Does it signpost peer learning, mentoring, programmes?
- Is it honest about module scope and purpose?

## Application Guidance

### For Module Designers
Use these questions during module design to test whether content genuinely addresses power, inclusion, and transformational learning. Select questions most relevant to the specific competency and stage. Ensure at least one question from each cross-cutting theme is considered.

### For Module Reviewers
Use these questions to evaluate draft modules. Look for evidence that the module addresses the questions, not just mentions diversity. Check whether examples, activities, and prompts genuinely engage with power and difference rather than treating them as add-ons.

### For Module Pilot Testers
Use personas and their evaluation questions to test modules from diverse perspectives. Ask: "If I were [persona], would this module work for me? Would it address my specific barriers? Would it help me develop this competency in my context?"

### For Quality Assurance
Use cross-cutting themes as quality standards. All modules should address Principle 10, provide transformational inquiry, be accessible, support contextualisation, guide evidence gathering, match level, integrate I-We-It, and acknowledge limitations.

### Iterative Development
These questions should be used throughout iterative design cycles, not as one-time checklist. Return to them repeatedly as modules develop. Use them to identify gaps, refine content, and ensure genuine inclusion rather than superficial diversity references.
