# Module Testing Criteria and Evaluation Questions
## Stage 1: New and First-Line Managers and Leaders

**Version:** 1.0

---

## Document Purpose

This document contains evaluation questions and testing scenarios for assessing whether learning modules genuinely address power, difference, inclusion, and transformational learning as specified in the design guidelines. These criteria are derived from the diverse personas across all stages of the framework and support gap-analysis testing of module design and content.

## Usage Guidance

Use these questions and scenarios to test modules against:
- Design Principle 10 (Difference, diversity, inclusion, and power as golden thread)
- P + Q = L formula (transformational inquiry not just information)
- I-We-It integration
- Level-appropriate complexity
- Accessibility and inclusion
- Evidence gathering support
- Acknowledgment of limitations

## Personas

### Persona 5: Senior Community Speech and Language Therapist
**Diversity Context:** White British woman, bisexual (not out at work), heteronormative assumptions

#### Module Evaluation Questions
- Does this module assume heterosexual norms in examples of relationships/family?
- Does it address creating inclusive environments for LGBTQ+ staff and patients?
- Will this help me navigate being authentic while managing a team?
- Does it recognise invisible labour of managing disclosure decisions?
- Does it help challenge heteronormative assumptions without outing myself?

#### Module Testing Scenarios

**Scenario 1: Manage conflict and sensitive conversations**
- **Test Question:** Does module provide practical frameworks for difficult conversations? Does it address the emotional dimension of accountability conversations?
- **Evaluation Criteria:**
  - Provides structured approaches to difficult conversations
  - Addresses emotional labour and anxiety
  - Balances support with accountability

**Scenario 2: Manage and measure performance**
- **Test Question:** Does module help me understand how to measure and improve performance in professional roles? Does it address transition from peer to manager?
- **Evaluation Criteria:**
  - Explains performance management in professional contexts
  - Addresses identity shift peer to manager
  - Offers strategies for measuring qualitative work

**Scenario 3: Communicate with clarity and purpose**
- **Test Question:** Does module address communication with team vs with patients? Does it help me develop authentic leadership voice?
- **Evaluation Criteria:**
  - Distinguishes managerial from clinical communication
  - Addresses authenticity and authority
  - Helps develop individual leadership voice

#### Module Evaluation Questions

- Does it recognise complexity of mixed heritage identities?
- Does it address colourism and privilege within racialised communities?

- Does it recognise emotional labour of navigating multiple identity expectations?

#### Module Testing Scenarios

- **Test Question:** Does module help build relationships across teams to address maternal health inequalities? Does it address navigating power dynamics when advocating for Black women's health?
- **Evaluation Criteria:**
  - Provides frameworks for cross-team relationship building
  - Addresses power dynamics in advocacy conversations
  - Helps build strategic partnerships to improve maternal outcomes
  - Recognises emotional labour of advocating while experiencing discrimination

- **Evaluation Criteria:**

  - Helps navigate being labelled "angry" or "difficult" when challenging practice
  - Recognises intersectionality in care needs

### Persona 7: Community Pharmacist
**Diversity Context:** British Pakistani woman, Muslim, disabled (wheelchair user - spinal cord injury)

#### Module Evaluation Questions
- Is this module accessible for disabled learners (captioning, screen reader compatible, flexible pacing)?
- Does it address ableism in healthcare leadership and progression?
- Does it recognise intersections of disability, race, and religion?
- Will this help me advocate for accessible workplaces and career paths?
- Does it challenge assumptions about disabled people's capability to lead?
- Does it explicitly address Islamophobia and religious discrimination?
- Do examples recognise hijab-wearing Muslim women face compound discrimination?
- Does it address how Ramadan and prayer times require workplace accommodation?
- Will this help me navigate stereotyping about Muslim women as submissive?
- Does it address security targeting and surveillance of Muslim staff?

#### Module Testing Scenarios

**Scenario 1: Build engagement**
- **Test Question:** Does module help build engagement across diverse teams? Does it address creating inclusive environments for disabled, Muslim, and minoritised staff?
- **Evaluation Criteria:**
  - Provides frameworks for inclusive team engagement
  - Addresses multiple intersecting barriers (disability, race, religion)
  - Helps create workplace culture that accommodates diverse needs
  - Challenges assumptions about "ideal worker" as able-bodied, white, non-religious

**Scenario 2: Lead a collaborative team**
- **Test Question:** Does module help lead collaborative teams across different professionals? Does it address accessibility and inclusion in team leadership?
- **Evaluation Criteria:**
  - Provides strategies for cross-professional team leadership
  - Addresses physical and attitudinal accessibility in team working
  - Helps navigate leading while experiencing compound discrimination
  - Values diverse leadership styles and approaches

**Scenario 3: Make sure people feel safe in the workplace**
- **Test Question:** Does module address creating psychologically and physically safe workplaces for disabled, Muslim, and minoritised staff? Does it help me address Islamophobia and ableism?
- **Evaluation Criteria:**
  - Addresses multiple dimensions of workplace safety (physical, psychological, cultural)
  - Provides frameworks for addressing discrimination and microaggressions
  - Helps create policies for religious accommodation and accessibility
  - Recognises compound impact of intersecting marginalisation
  - Values reporting and addressing safety concerns as leadership responsibility

## Cross-Cutting Evaluation Themes

### Theme 1: Principle 10 - Power, Difference, and Inclusion
**Key Questions:**
- Does the module treat diversity as integral to competency, not separate topic?
- Does it surface who has power, whose knowledge is privileged, whose voices are heard?
- Does it address structural inequalities, not just individual behaviour?
- Does it help participants recognise their own positionality and privilege?
- Does it provide frameworks for action, not just awareness?

### Theme 2: Transformational Inquiry (Q component)
**Key Questions:**
- Does the module provoke genuine questioning of assumptions?
- Does it create discomfort that leads to transformed understanding?
- Is there space for critical reflection on power and practice?
- Does it challenge rather than confirm existing thinking?
- Does it move beyond information transfer to transformation?

### Theme 3: Accessibility and Universal Design
**Key Questions:**
- Does the module work for diverse learning needs and access requirements?
- Are examples and language accessible across educational backgrounds?
- Does it assume standard working patterns and availability?
- Can it be accessed flexibly across time zones, shifts, caring responsibilities?
- Does it provide multiple entry points and formats?

### Theme 4: Contextualisation and Application
**Key Questions:**
- Does the module prompt participants to translate to their specific context?
- Does it recognise diverse NHS roles, settings, and contexts?
- Does it address practical constraints and systemic pressures?
- Does it avoid generic examples that assume particular contexts?
- Does it help participants navigate their unique situation?

### Theme 5: Evidence Gathering and Competency Assessment
**Key Questions:**
- Does the module clearly guide what evidence demonstrates competency?
- Does it acknowledge completion â‰  competency achievement?
- Does it provide realistic timeframes for evidence gathering?
- Does it direct toward formal assessment processes?
- Does it support ongoing practice-based evidence collection?

### Theme 6: Level-Appropriate Complexity
**Key Questions:**
- Is conceptual complexity appropriate for the stage?
- Does scope match organisational perspective (team/service/system)?
- Is balance of guidance vs exploration appropriate?
- Does it respect developmental readiness?
- Does it avoid patronising or overwhelming?

### Theme 7: Integration Across I-We-It
**Key Questions:**
- Does module address personal mastery (I) dimension?
- Does it address relational capability (We) dimension?
- Does it address operational effectiveness (It) dimension?
- Does it show how these integrate rather than separate?
- Does it avoid fragmented skill development?

### Theme 8: Acknowledgment of Limitations
**Key Questions:**
- Does module explicitly state what it cannot achieve?
- Does it acknowledge 2-hour engagement cannot complete development?
- Does it direct toward deeper learning opportunities?
- Does it signpost peer learning, mentoring, programmes?
- Is it honest about module scope and purpose?

## Application Guidance

### For Module Designers
Use these questions during module design to test whether content genuinely addresses power, inclusion, and transformational learning. Select questions most relevant to the specific competency and stage. Ensure at least one question from each cross-cutting theme is considered.

### For Module Reviewers
Use these questions to evaluate draft modules. Look for evidence that the module addresses the questions, not just mentions diversity. Check whether examples, activities, and prompts genuinely engage with power and difference rather than treating them as add-ons.

### For Module Pilot Testers
Use personas and their evaluation questions to test modules from diverse perspectives. Ask: "If I were [persona], would this module work for me? Would it address my specific barriers? Would it help me develop this competency in my context?"

### For Quality Assurance
Use cross-cutting themes as quality standards. All modules should address Principle 10, provide transformational inquiry, be accessible, support contextualisation, guide evidence gathering, match level, integrate I-We-It, and acknowledge limitations.

### Iterative Development
These questions should be used throughout iterative design cycles, not as one-time checklist. Return to them repeatedly as modules develop. Use them to identify gaps, refine content, and ensure genuine inclusion rather than superficial diversity references.
