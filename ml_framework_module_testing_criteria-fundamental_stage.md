# Module Testing Criteria and Evaluation Questions
## Fundamental Stage

**Version:** 1.0

---

## Document Purpose

This document contains evaluation questions and testing scenarios for assessing whether learning modules genuinely address power, difference, inclusion, and transformational learning as specified in the design guidelines. These criteria are derived from the diverse personas across all stages of the framework and support gap-analysis testing of module design and content.

## Usage Guidance

Use these questions and scenarios to test modules against:
- Design Principle 10 (Difference, diversity, inclusion, and power as golden thread)
- P + Q = L formula (transformational inquiry not just information)
- I-We-It integration
- Level-appropriate complexity
- Accessibility and inclusion
- Evidence gathering support
- Acknowledgment of limitations

## Personas

### Persona 1: Medical Secretary
**Diversity Context:** White British woman, working class background, single parent, non-graduate

#### Module Evaluation Questions
- Does this module recognise that class and educational background shape progression opportunities?
- Do examples assume everyone has degree-level education and professional family background?
- Does it address practical barriers like childcare, finances, and time?
- Will this help me challenge assumptions about non-clinical staff capabilities?
- Does it recognise the emotional labour of administrative roles?

#### Module Testing Scenarios

**Scenario 1: Communicate with clarity and purpose**
- **Test Question:** Does module address communication across power differentials (admin to clinical staff)? Does it recognise accent/class markers affect how communication is received?
- **Evaluation Criteria:**
  - Acknowledges hierarchies in NHS settings
  - Addresses communication barriers beyond "technique"
  - Includes examples from non-clinical perspectives

**Scenario 2: Take accountability for my actions**
- **Test Question:** Does module help me understand accountability when I have responsibility but limited formal authority?
- **Evaluation Criteria:**
  - Distinguishes between authority and accountability
  - Addresses accountability in matrix working
  - Recognises constraints on agency

**Scenario 3: Build engagement**
- **Test Question:** Does module recognise the invisible labour of engagement work? Does it address building engagement from non-leadership positions?
- **Evaluation Criteria:**
  - Values informal leadership and relationship work
  - Addresses gender/class assumptions about care work
  - Practical strategies for limited resources

### Persona 2: Emergency Care Assistant
**Diversity Context:** South Asian (Pakistani) man, Muslim, working class, experiences Islamophobia

#### Module Evaluation Questions
- Does this recognise religious needs as legitimate workplace requirement?
- Do examples assume white, secular, Christian cultural norms as default?
- Does it help me address Islamophobic "banter" while maintaining relationships?
- Will this help colleagues understand why representation matters?
- Does it address the emotional labour of managing others' prejudice?

#### Module Testing Scenarios

**Scenario 1: Manage with civility and compassion**
- **Test Question:** Does module address managing patient prejudice while maintaining compassionate care? Does it recognise religious/cultural dimensions of compassion?
- **Evaluation Criteria:**
  - Acknowledges staff as recipients of incivility/discrimination
  - Addresses maintaining professional boundaries with abusive patients
  - Recognises diverse cultural expressions of care

**Scenario 2: Manage challenges**
- **Test Question:** Does module address systemic challenges (underfunding, violence) vs individual challenges? Does it help me distinguish what's in my control?
- **Evaluation Criteria:**
  - Distinguishes systemic from interpersonal challenges
  - Addresses moral injury when system constrains good care
  - Provides frameworks for action within constraints

**Scenario 3: Communicate with clarity and purpose**
- **Test Question:** Does module address code-switching and communication across cultural contexts? Does it help me challenge discriminatory communication?
- **Evaluation Criteria:**
  - Recognises diverse communication styles as valid
  - Addresses power in communication (who gets heard)
  - Helps navigate challenging discriminatory language

### Persona 3: Health Care Assistant
**Diversity Context:** White British woman, disabled (chronic pain, anxiety), part-time, career break

#### Module Evaluation Questions
- Does this module recognise that IT/system barriers are disability equity issues?
- Do examples assume able-bodied, neurotypical, full-time working patterns?
- Will this help me advocate for reasonable adjustments without being seen as problematic?
- Does it address managing invisible disability in demanding environments?
- Does it treat access needs as legitimate requirement not special favour?

#### Module Testing Scenarios

**Scenario 1: Develop personal health and wellbeing strategies**
- **Test Question:** Does module recognise managing chronic health conditions while working in demanding healthcare environment? Does it address reasonable adjustments as enabling rather than limiting?
- **Evaluation Criteria:**
  - Addresses disability/chronic illness in workforce
  - Discusses sustainability and pacing strategies
  - Frames adjustments as enabling excellence

**Scenario 2: Use data, evidence and critical thinking**
- **Test Question:** Does module address IT accessibility and cognitive load? Does it recognise that digital literacy is learned not innate?
- **Evaluation Criteria:**
  - Acknowledges IT barriers as systemic issue
  - Provides accessible learning strategies
  - Validates different learning speeds/needs

**Scenario 3: Commit to continuing professional development**
- **Test Question:** Does module recognise part-time work, caring responsibilities, and health constraints as legitimate? Does it frame CPD flexibly?
- **Evaluation Criteria:**
  - Offers flexible approaches to CPD
  - Validates non-traditional career paths
  - Addresses structural barriers to development

### Persona 4: Community Therapy Support Worker
**Diversity Context:** Black British (Nigerian) woman, aspiring professional, shift work constraints

#### Module Evaluation Questions
- Does this module acknowledge systemic barriers to professional development for Black staff?
- Do examples assume traditional educational pathways and full-time study?
- Will this help me navigate professional development when supervisor availability limited?
- Does it address managing patient racism while maintaining therapeutic relationships?
- Does it validate non-traditional routes to professional qualification?

#### Module Testing Scenarios

**Scenario 1: Commit to continuing professional development**
- **Test Question:** Does module acknowledge structural barriers (shift work, cost, supervision access) to CPD? Does it address navigating development when traditional support structures unavailable?
- **Evaluation Criteria:**
  - Recognises systemic barriers to development
  - Offers strategies for advocating for learning needs
  - Addresses flexible approaches when traditional routes blocked

**Scenario 2: Build relationships**
- **Test Question:** Does module address power dynamics in supervisory relationships? Does it help me navigate building relationships across hierarchies when I have less power?
- **Evaluation Criteria:**
  - Addresses power in professional relationships
  - Offers strategies for relationship-building from less powerful positions
  - Recognises emotional labour of managing across difference

**Scenario 3: Build engagement**
- **Test Question:** Does module address engaging when shifts misaligned? Does it recognise challenges of 7-day service and diverse shift patterns?
- **Evaluation Criteria:**
  - Offers practical strategies for asynchronous engagement
  - Addresses building team connection despite shift diversity
  - Recognises structural constraints on engagement

## Cross-Cutting Evaluation Themes

### Theme 1: Principle 10 - Power, Difference, and Inclusion
**Key Questions:**
- Does the module treat diversity as integral to competency, not separate topic?
- Does it surface who has power, whose knowledge is privileged, whose voices are heard?
- Does it address structural inequalities, not just individual behaviour?
- Does it help participants recognise their own positionality and privilege?
- Does it provide frameworks for action, not just awareness?

### Theme 2: Transformational Inquiry (Q component)
**Key Questions:**
- Does the module provoke genuine questioning of assumptions?
- Does it create discomfort that leads to transformed understanding?
- Is there space for critical reflection on power and practice?
- Does it challenge rather than confirm existing thinking?
- Does it move beyond information transfer to transformation?

### Theme 3: Accessibility and Universal Design
**Key Questions:**
- Does the module work for diverse learning needs and access requirements?
- Are examples and language accessible across educational backgrounds?
- Does it assume standard working patterns and availability?
- Can it be accessed flexibly across time zones, shifts, caring responsibilities?
- Does it provide multiple entry points and formats?

### Theme 4: Contextualisation and Application
**Key Questions:**
- Does the module prompt participants to translate to their specific context?
- Does it recognise diverse NHS roles, settings, and contexts?
- Does it address practical constraints and systemic pressures?
- Does it avoid generic examples that assume particular contexts?
- Does it help participants navigate their unique situation?

### Theme 5: Evidence Gathering and Competency Assessment
**Key Questions:**
- Does the module clearly guide what evidence demonstrates competency?
- Does it acknowledge completion â‰  competency achievement?
- Does it provide realistic timeframes for evidence gathering?
- Does it direct toward formal assessment processes?
- Does it support ongoing practice-based evidence collection?

### Theme 6: Level-Appropriate Complexity
**Key Questions:**
- Is conceptual complexity appropriate for the stage?
- Does scope match organisational perspective (team/service/system)?
- Is balance of guidance vs exploration appropriate?
- Does it respect developmental readiness?
- Does it avoid patronising or overwhelming?

### Theme 7: Integration Across I-We-It
**Key Questions:**
- Does module address personal mastery (I) dimension?
- Does it address relational capability (We) dimension?
- Does it address operational effectiveness (It) dimension?
- Does it show how these integrate rather than separate?
- Does it avoid fragmented skill development?

### Theme 8: Acknowledgment of Limitations
**Key Questions:**
- Does module explicitly state what it cannot achieve?
- Does it acknowledge 2-hour engagement cannot complete development?
- Does it direct toward deeper learning opportunities?
- Does it signpost peer learning, mentoring, programmes?
- Is it honest about module scope and purpose?

## Application Guidance

### For Module Designers
Use these questions during module design to test whether content genuinely addresses power, inclusion, and transformational learning. Select questions most relevant to the specific competency and stage. Ensure at least one question from each cross-cutting theme is considered.

### For Module Reviewers
Use these questions to evaluate draft modules. Look for evidence that the module addresses the questions, not just mentions diversity. Check whether examples, activities, and prompts genuinely engage with power and difference rather than treating them as add-ons.

### For Module Pilot Testers
Use personas and their evaluation questions to test modules from diverse perspectives. Ask: "If I were [persona], would this module work for me? Would it address my specific barriers? Would it help me develop this competency in my context?"

### For Quality Assurance
Use cross-cutting themes as quality standards. All modules should address Principle 10, provide transformational inquiry, be accessible, support contextualisation, guide evidence gathering, match level, integrate I-We-It, and acknowledge limitations.

### Iterative Development
These questions should be used throughout iterative design cycles, not as one-time checklist. Return to them repeatedly as modules develop. Use them to identify gaps, refine content, and ensure genuine inclusion rather than superficial diversity references.
